Cornell Notes: การออกแบบและเทคโนโลยี ม.4

หนังสือเรียนรายวิชาพื้นฐานวิทยาศาสตร์และเทคโนโลยี (สสวท.)
ฉบับเต็ม ตามสารบัญ 7 บท
1
ระบบทางเทคโนโลยีที่ซับซ้อน
1.1 ระบบทางเทคโนโลยีคืออะไร
ระบบคืออะไร
องค์ประกอบของระบบมีอะไรบ้าง
Feedback สำคัญอย่างไร
ระบบ (System) คือ กลุ่มของส่วนประกอบที่ทำงานร่วมกันอย่างเป็นระเบียบเพื่อให้บรรลุเป้าหมายที่กำหนด
องค์ประกอบหลัก 4 ส่วน:
Input (ตัวป้อน): สิ่งที่ป้อนเข้าสู่ระบบ เช่น วัตถุดิบ พลังงาน ข้อมูล
Process (กระบวนการ): ขั้นตอนการแปลง Input ให้เป็น Output
Output (ผลผลิต): ผลลัพธ์ที่ได้จากกระบวนการ
Feedback (ข้อมูลย้อนกลับ): ข้อมูลที่ส่งกลับเพื่อปรับปรุงระบบ
Feedback ทำให้ระบบปรับตัวเองได้ เช่น เครื่องปรับอากาศรับ Feedback จากอุณหภูมิห้องแล้วปรับความเย็น
ตัวอย่าง: พัดลม → Input = ไฟฟ้า, Process = มอเตอร์หมุนใบพัด, Output = ลม
1.2 ระบบทางเทคโนโลยีที่ซับซ้อน
ระบบซับซ้อนต่างจากระบบธรรมดาอย่างไร
ระบบย่อยเชื่อมกันอย่างไร
ตัวอย่างระบบซับซ้อนมีอะไร
ระบบซับซ้อน ประกอบด้วยระบบย่อย (Subsystem) หลายระบบทำงานสัมพันธ์กัน
Output ของระบบย่อยหนึ่ง อาจเป็น Input ของอีกระบบย่อยหนึ่ง (เชื่อมโยงแบบลูกโซ่)
ตัวอย่างระบบซับซ้อน:
รถยนต์ = ระบบเครื่องยนต์ + ระบบเบรก + ระบบส่งกำลัง + ระบบไฟฟ้า + ระบบระบายความร้อน
สมาร์ทโฟน = ระบบจอสัมผัส + ระบบประมวลผล + ระบบสื่อสาร + ระบบกล้อง + ระบบแบตเตอรี่
โรงงานผลิต = ระบบจัดเก็บวัตถุดิบ + ระบบผลิต + ระบบตรวจสอบ + ระบบขนส่ง
ยิ่งระบบซับซ้อน ยิ่งต้องออกแบบให้แต่ละส่วนประสานงานกันดี
1.3 ระบบทางเทคโนโลยีที่ทำงานผิดพลาด
สาเหตุที่ระบบผิดพลาด
ผิดพลาดแล้วเกิดอะไรขึ้น
จะป้องกันอย่างไร
สาเหตุที่ระบบทำงานผิดพลาด:
1. ส่วนประกอบเสื่อมสภาพ/ชำรุด
2. Input ไม่ถูกต้อง/ไม่เพียงพอ
3. การใช้งานผิดวิธี (Human Error)
4. สภาพแวดล้อมไม่เหมาะสม (ความชื้น, อุณหภูมิ)
เมื่อระบบย่อยใดผิดพลาด จะส่งผลกระทบเป็นลูกโซ่ไปทั้งระบบ
เช่น ระบบระบายความร้อนรถยนต์พัง → เครื่องยนต์ร้อนเกิน → เครื่องดับ
การป้องกัน: บำรุงรักษาตามกำหนด, ตรวจสอบ Feedback สม่ำเสมอ, ออกแบบระบบสำรอง
1.4 การแก้ไขข้อบกพร่องของระบบ
วิเคราะห์ข้อบกพร่องอย่างไร
ขั้นตอนแก้ไขมีอะไร
ขั้นตอนการวิเคราะห์และแก้ไข:
1. ระบุอาการ: ระบบแสดงอาการผิดปกติอะไร
2. วิเคราะห์สาเหตุ: ตรวจสอบระบบย่อยทีละส่วน หาจุดที่เป็นปัญหา
3. เลือกวิธีแก้: ซ่อมแซม เปลี่ยนชิ้นส่วน หรือออกแบบใหม่
4. ดำเนินการแก้ไข: ลงมือแก้และทดสอบ
5. ตรวจสอบผล: ระบบกลับมาทำงานปกติหรือไม่
ใช้แนวคิด "แยกส่วน" (Decomposition) เพื่อหาจุดบกพร่องได้ง่ายขึ้น
สรุปบทที่ 1
เทคโนโลยีทำงานเป็นระบบ (Input → Process → Output → Feedback) ระบบซับซ้อนคือหลายระบบย่อยเชื่อมกัน ถ้าระบบย่อยใดพัง จะกระทบทั้งหมดแบบลูกโซ่ แก้โดยวิเคราะห์แยกส่วน หาจุดบกพร่อง แล้วซ่อม/ออกแบบใหม่
2
การเปลี่ยนแปลงของเทคโนโลยี
2.1 สาเหตุหรือปัจจัยการเปลี่ยนแปลงของเทคโนโลยี
ทำไมเทคโนโลยีต้องเปลี่ยน
ปัจจัยหลักมีกี่ข้อ
ปัจจัยภายในกับภายนอกต่างกันอย่างไร
เทคโนโลยีไม่คงที่ เปลี่ยนแปลงตลอดเวลาเพราะความต้องการของมนุษย์ไม่หยุดนิ่ง
ปัจจัยที่ทำให้เทคโนโลยีเปลี่ยนแปลง:
1. ปัญหาหรือความต้องการของมนุษย์: ต้องการความสะดวก ปลอดภัย รวดเร็ว ประหยัดมากขึ้น
2. ความก้าวหน้าทางวิทยาศาสตร์: ค้นพบวัสดุใหม่ ทฤษฎีใหม่ ทำให้พัฒนาเทคโนโลยีใหม่ได้
3. เศรษฐกิจ: ต้นทุนการผลิต การแข่งขัน ความต้องการตลาด
4. สังคมและวัฒนธรรม: ค่านิยม ไลฟ์สไตล์ที่เปลี่ยนไป
5. สิ่งแวดล้อม: ปัญหามลพิษ ภาวะโลกร้อน ผลักดันเทคโนโลยีสีเขียว
6. นโยบายรัฐและกฎหมาย: กฎระเบียบความปลอดภัย มาตรฐาน
แบ่งเป็น: ปัจจัยภายใน (ข้อจำกัดของตัวเทคโนโลยีเอง) กับ ปัจจัยภายนอก (สังคม เศรษฐกิจ กฎหมาย)
2.2 ตัวอย่างการเปลี่ยนแปลงของเทคโนโลยี
การเปลี่ยนแปลงมีรูปแบบอะไร
ตัวอย่างเทคโนโลยีที่เปลี่ยนแปลง
เส้นเวลาใช้ทำอะไร
รูปแบบการเปลี่ยนแปลง:
ปรับปรุง (Improvement): ทำของเดิมให้ดีขึ้น เร็วขึ้น ถูกลง
นวัตกรรม (Innovation): สร้างสิ่งใหม่ที่ไม่เคยมีมาก่อน
ตัวอย่างการเปลี่ยนแปลง:
โทรศัพท์: บ้าน → มือถือ → สมาร์ทโฟน → โทรศัพท์พับได้
การสื่อสาร: จดหมาย → โทรเลข → อีเมล → แชท → วิดีโอคอล
แสงสว่าง: เทียน → หลอดไส้ → หลอดฟลูออเรสเซนต์ → LED
การเดินทาง: เดินเท้า → ม้า → รถยนต์ → รถไฟฟ้า → รถไร้คนขับ
เส้นเวลาเทคโนโลยี (Technology Timeline) ใช้วิเคราะห์ว่าเทคโนโลยีเปลี่ยนอย่างไรในแต่ละยุค
สังเกต: ยิ่งยุคหลัง เทคโนโลยียิ่งเปลี่ยนเร็วขึ้น (Exponential Growth)
สรุปบทที่ 2
เทคโนโลยีเปลี่ยนเพราะปัจจัยหลายด้าน (ความต้องการ, วิทยาศาสตร์, เศรษฐกิจ, สังคม, สิ่งแวดล้อม, กฎหมาย) โดยเปลี่ยนแบบปรับปรุงหรือสร้างนวัตกรรมใหม่ ใช้เส้นเวลาวิเคราะห์วิวัฒนาการ ยิ่งปัจจุบันยิ่งเปลี่ยนเร็ว
3
ผลกระทบของเทคโนโลยี
3.1 ผลกระทบของเทคโนโลยีต่อมนุษย์
ผลกระทบด้านบวกต่อมนุษย์
ผลกระทบด้านลบต่อมนุษย์
ด้านบวก:
อำนวยความสะดวกในชีวิตประจำวัน, ทำงานเร็วขึ้น
รักษาโรคได้ดีขึ้น, อายุยืนขึ้น
เข้าถึงข้อมูลและการศึกษาได้ง่าย
ด้านลบ:
ปัญหาสุขภาพ: สายตาเสีย, ออฟฟิศซินโดรม, นอนดึก
พึ่งพาเทคโนโลยีมากเกินไป ขาดทักษะชีวิต
ข้อมูลส่วนตัวถูกละเมิด, ภัยไซเบอร์
3.2 ผลกระทบของเทคโนโลยีต่อสังคมและเศรษฐกิจ
ผลกระทบต่อสังคม
ผลกระทบต่อเศรษฐกิจ
ต่อสังคม:
บวก: เชื่อมต่อผู้คนทั่วโลก, แบ่งปันความรู้, ลดอุปสรรคด้านระยะทาง
ลบ: ช่องว่างทางดิจิทัล (Digital Divide), ข่าวปลอม, การกลั่นแกล้งออนไลน์, แตกแยกในสังคม
ต่อเศรษฐกิจ:
บวก: สร้างอาชีพใหม่ (โปรแกรมเมอร์, YouTuber), ผลิตภาพสูงขึ้น, e-commerce ขยายตลาด
ลบ: แรงงานถูกแทนที่ด้วยเครื่องจักร/AI, ธุรกิจเก่าล้มหายตายจาก
3.3 ผลกระทบของเทคโนโลยีต่อสิ่งแวดล้อม
ผลกระทบบวกต่อสิ่งแวดล้อม
ผลกระทบลบต่อสิ่งแวดล้อม
ด้านบวก:
พลังงานสะอาด (โซลาร์เซลล์, กังหันลม), รีไซเคิล, ลดการใช้กระดาษ
ติดตามสภาพอากาศ/มลพิษแบบเรียลไทม์
ด้านลบ:
ขยะอิเล็กทรอนิกส์ (e-waste) ย่อยสลายยาก, สารเคมีอันตราย
การผลิตปล่อยก๊าซเรือนกระจก, ใช้ทรัพยากรธรรมชาติมากขึ้น
การทำเหมืองแร่หายากสำหรับอุปกรณ์อิเล็กทรอนิกส์
3.4 ตัวอย่างการวิเคราะห์ผลกระทบของเทคโนโลยี
วิเคราะห์ผลกระทบอย่างไร
ใช้หลักอะไรในการวิเคราะห์
ขั้นตอนวิเคราะห์ผลกระทบ:
1. ระบุเทคโนโลยีที่จะวิเคราะห์
2. วิเคราะห์ผลกระทบแยกทั้ง 4 ด้าน (มนุษย์, สังคม, เศรษฐกิจ, สิ่งแวดล้อม)
3. แยกผลบวกและลบในแต่ละด้าน
4. ระบุผู้ได้ประโยชน์และผู้เสียประโยชน์
5. พิจารณาผลกระทบระยะสั้นและระยะยาว
6. สรุปและเสนอแนวทางลดผลกระทบด้านลบ
ตัวอย่าง: วิเคราะห์ผลกระทบของ "สมาร์ทโฟน" หรือ "รถยนต์ไฟฟ้า" ต่อ 4 ด้าน
ใช้ตาราง 4 ช่อง (ด้าน × บวก/ลบ) ช่วยจัดระเบียบการวิเคราะห์
สรุปบทที่ 3
เทคโนโลยีทุกอย่างมีผลกระทบทั้งบวกและลบต่อ 4 ด้าน (มนุษย์, สังคม, เศรษฐกิจ, สิ่งแวดล้อม) ต้องวิเคราะห์รอบด้านก่อนนำมาใช้ โดยพิจารณาทั้งระยะสั้น/ยาว และหาทางลดผลเสีย
4
วัสดุและเครื่องมือพื้นฐาน
4.1 วัสดุ
วัสดุแบ่งเป็นกี่ประเภท
สมบัติของวัสดุมีอะไร
เลือกวัสดุอย่างไร
ประเภทวัสดุ:
โลหะ: เหล็ก, อะลูมิเนียม, ทองแดง → แข็งแรง นำไฟฟ้า นำความร้อน
พอลิเมอร์/พลาสติก: PE, PP, PVC, อะคริลิก → เบา ขึ้นรูปง่าย กันน้ำ
ไม้: ไม้สัก ไม้ยาง ไม้อัด MDF → ตัดง่าย น้ำหนักเบา
เซรามิก: กระเบื้อง แก้ว กระดาษ → ทนความร้อน แต่เปราะ
วัสดุผสม (Composite): คาร์บอนไฟเบอร์, ไฟเบอร์กลาส → รวมข้อดีหลายวัสดุ
สมบัติของวัสดุ ที่ต้องพิจารณา:
ความแข็ง (Hardness): ทนต่อการขูดขีด
ความเหนียว (Toughness): ทนต่อแรงกระแทก ไม่แตกง่าย
ความยืดหยุ่น (Elasticity): ยืดแล้วกลับรูปเดิม
การนำไฟฟ้า/ความร้อน, ทนการกัดกร่อน, ความหนาแน่น
เลือกวัสดุตาม: สมบัติที่ต้องการ + ต้นทุน + ความหาง่าย + ผลกระทบต่อสิ่งแวดล้อม
4.2 เครื่องมือพื้นฐาน
เครื่องมือแบ่งเป็นกี่กลุ่ม
ข้อควรระวังในการใช้
เครื่องมือพื้นฐานแบ่งตามหน้าที่:
เครื่องมือวัด: ไม้บรรทัด, ตลับเมตร, เวอร์เนียร์, ฉาก → ใช้วัดขนาด มุม ระยะ
เครื่องมือตัด: เลื่อยมือ, เลื่อยฉลุ, กรรไกร, คัตเตอร์ → ตัดวัสดุให้ได้รูปร่าง
เครื่องมือเจาะ: สว่านมือ, สว่านไฟฟ้า, ดอกสว่าน → เจาะรูในวัสดุ
เครื่องมือยึด: ค้อน, ไขควง, ประแจ, คีมล็อก → ยึดต่อชิ้นส่วน
เครื่องมือตกแต่ง: กระดาษทราย, ตะไบ, กบ → ทำผิวเรียบ
ข้อควรระวัง:
สวมอุปกรณ์ป้องกัน (แว่น, ถุงมือ), ใช้เครื่องมือให้ถูกประเภท, เก็บให้เป็นที่
4.3 การตัด ต่อ และขึ้นรูปวัสดุ
การตัดวัสดุมีกี่วิธี
การต่อวัสดุมีแบบไหนบ้าง
การขึ้นรูปคืออะไร
การตัด (Cutting):
เลื่อย: ตัดไม้, พลาสติก, โลหะบาง
คัตเตอร์: ตัดกระดาษ, ฟิวเจอร์บอร์ด, พลาสติกบาง
กรรไกร: ตัดแผ่นวัสดุบาง
เลเซอร์/CNC: ตัดแม่นยำสูง (เทคโนโลยีสมัยใหม่)
การต่อ (Joining):
ถาวร: กาว, บัดกรี, เชื่อม, ตะปู
ชั่วคราว: สกรู, น็อต, สลัก (ถอดประกอบได้)
การขึ้นรูป (Forming):
พับ/ดัด: ใช้กับโลหะแผ่น พลาสติก
หล่อ/เท: เทวัสดุเหลวลงแม่พิมพ์ (โลหะ, พลาสเตอร์)
ปั๊ม/กด: อัดวัสดุให้เป็นรูปร่างด้วยแรงกด
พิมพ์ 3D (3D Printing): สร้างชิ้นงานทีละชั้น จากไฟล์ดิจิทัล
สรุปบทที่ 4
รู้จัก 5 ประเภทวัสดุ (โลหะ, พอลิเมอร์, ไม้, เซรามิก, ผสม) และสมบัติของแต่ละชนิด เลือกใช้ให้เหมาะกับงาน รู้จักเครื่องมือพื้นฐาน 5 กลุ่ม และเทคนิคตัด ต่อ ขึ้นรูป เพื่อสร้างชิ้นงานจริง
5
กลไก ไฟฟ้า และอิเล็กทรอนิกส์
5.1 กลไก
กลไกคืออะไร
กลไกพื้นฐานมีกี่แบบ
แต่ละแบบใช้ทำอะไร
กลไก (Mechanism) คือ ชิ้นส่วนที่ทำงานร่วมกันเพื่อเปลี่ยนทิศทาง ขนาด หรือชนิดของการเคลื่อนที่/แรง
กลไกพื้นฐาน:
เฟือง (Gear): เปลี่ยนทิศทางการหมุน, เพิ่ม/ลดความเร็ว (อัตราทด)
เฟืองเล็กหมุนเร็ว แรงน้อย / เฟืองใหญ่หมุนช้า แรงมาก
รอก (Pulley): ผ่อนแรง, เปลี่ยนทิศทางแรง
รอกเดี่ยวเปลี่ยนทิศทาง / รอกพวงผ่อนแรง
คาน (Lever): ผ่อนแรง ประกอบด้วย จุดหมุน, แรงพยายาม, แรงต้านทาน
3 ประเภท: จุดหมุนอยู่กลาง / แรงต้านอยู่กลาง / แรงพยายามอยู่กลาง
ลูกเบี้ยว (Cam): เปลี่ยนการหมุนเป็นการเคลื่อนที่เชิงเส้น (ขึ้น-ลง)
ข้อเหวี่ยง (Crank): เปลี่ยนการหมุนเป็นการเคลื่อนที่ไป-กลับ
ตัวอย่าง: จักรยาน (เฟือง+โซ่), กรรไกร (คาน), ธงชาติขึ้นเสา (รอก)
5.2 อุปกรณ์ไฟฟ้าและอิเล็กทรอนิกส์
อุปกรณ์ไฟฟ้ากับอิเล็กทรอนิกส์ต่างกันอย่างไร
อุปกรณ์พื้นฐานมีอะไรบ้าง
วงจรไฟฟ้าเบื้องต้น
อุปกรณ์ไฟฟ้า: ใช้พลังงานไฟฟ้าโดยตรง
มอเตอร์ (เปลี่ยนไฟฟ้าเป็นการหมุน), หลอดไฟ, สวิตช์, แบตเตอรี่, ฟิวส์
อุปกรณ์อิเล็กทรอนิกส์: ควบคุมกระแสไฟฟ้าขนาดเล็ก
LED (ไดโอดเปล่งแสง): ใช้ไฟน้อย อายุยืน แสดงสถานะ
ตัวต้านทาน (Resistor): จำกัดกระแสไฟฟ้า
ตัวเก็บประจุ (Capacitor): เก็บพลังงานชั่วคราว
ทรานซิสเตอร์: เป็นสวิตช์อิเล็กทรอนิกส์ / ขยายสัญญาณ
เซนเซอร์: ตรวจจับแสง (LDR), อุณหภูมิ, เสียง, การเคลื่อนไหว
วงจรไฟฟ้าพื้นฐาน: แหล่งจ่ายไฟ → สายไฟ → โหลด (อุปกรณ์) → กลับแหล่งจ่าย
วงจรอนุกรม: ต่อเรียง (กระแสเท่ากัน, แรงดันแบ่ง)
วงจรขนาน: ต่อคู่ (แรงดันเท่ากัน, กระแสแบ่ง)
5.3 แผงควบคุมขนาดเล็ก
แผงควบคุมขนาดเล็กคืออะไร
ตัวอย่างบอร์ดที่ใช้
ทำงานอย่างไร
แผงควบคุมขนาดเล็ก (Microcontroller Board) คือ คอมพิวเตอร์ขนาดเล็กที่โปรแกรมสั่งงานได้
ตัวอย่างบอร์ดที่นิยม:
Arduino, micro:bit, ESP32, Raspberry Pi Pico
หลักการทำงาน:
Input (เซนเซอร์) → Process (ประมวลผลตามโปรแกรม) → Output (สั่งงานอุปกรณ์)
ตัวอย่างการใช้งาน:
เซนเซอร์แสง + micro:bit → ถ้ามืดให้เปิด LED อัตโนมัติ
เซนเซอร์อุณหภูมิ + Arduino → ถ้าร้อนให้เปิดพัดลม
เขียนโปรแกรมด้วย Block-based (ลากบล็อก) หรือ Text-based (เขียนโค้ด เช่น C/Python)
สรุปบทที่ 5
กลไก (เฟือง, รอก, คาน, ลูกเบี้ยว) ใช้เปลี่ยนแรงและการเคลื่อนที่ อุปกรณ์ไฟฟ้า/อิเล็กทรอนิกส์ทำให้ระบบทำงานอัตโนมัติ แผงควบคุมขนาดเล็ก (microcontroller) เป็นสมองกลที่เชื่อมเซนเซอร์กับอุปกรณ์ Output ผ่านโปรแกรม
6
กระบวนการออกแบบเชิงวิศวกรรม
6.1 ขั้นระบุปัญหา
ทำไมต้องระบุปัญหาให้ชัด
ต้องกำหนดอะไรบ้าง
ขั้นแรกและสำคัญที่สุด: ถ้าระบุปัญหาผิด ทุกขั้นตอนต่อไปจะผิดหมด
สิ่งที่ต้องทำ:
1. วิเคราะห์สถานการณ์ปัญหาให้ชัดเจน (ปัญหาคืออะไร เกิดกับใคร ที่ไหน)
2. กำหนดเงื่อนไข (Criteria): สิ่งที่ต้องทำได้ ผลลัพธ์ที่ต้องการ
3. กำหนดข้อจำกัด (Constraints): เวลา งบประมาณ วัสดุที่หาได้ ขนาด
ตัวอย่าง: "ออกแบบที่วางโทรศัพท์สำหรับใช้บนโต๊ะเรียน" → เงื่อนไข: รองรับโทรศัพท์ได้, ปรับมุมได้ / ข้อจำกัด: ใช้วัสดุไม่เกิน 50 บาท, ทำเสร็จใน 2 ชั่วโมง
6.2 ขั้นรวบรวมข้อมูลและแนวคิดที่เกี่ยวข้องกับปัญหา
ต้องรวบรวมข้อมูลอะไร
หาข้อมูลจากที่ไหน
ข้อมูลที่ต้องรวบรวม:
1. ความรู้ทางวิทยาศาสตร์/คณิตศาสตร์ที่เกี่ยวข้อง
2. เทคโนโลยีหรือผลิตภัณฑ์ที่มีอยู่แล้ว (เคยมีใครแก้ปัญหานี้อย่างไร)
3. สมบัติวัสดุที่อาจนำมาใช้
4. ความต้องการของผู้ใช้จริง
แหล่งข้อมูล: หนังสือ, อินเทอร์เน็ต, ผู้เชี่ยวชาญ, สังเกตการณ์, สัมภาษณ์ผู้ใช้
ใช้ Brainstorming ระดมความคิด ไม่ตัดสินในขั้นนี้
6.3 ขั้นออกแบบวิธีการแก้ปัญหา
ออกแบบอย่างไร
ต้องร่างกี่แบบ
เลือกแบบอย่างไร
ขั้นตอนการออกแบบ:
1. ร่างแบบ (Sketch) อย่างน้อย 2-3 แนวทาง ที่แตกต่างกัน
2. วาดภาพแบบร่าง 3 มิติ (Isometric) หรือภาพฉาย (Orthographic)
3. ระบุขนาด วัสดุ ส่วนประกอบในแบบ
4. เปรียบเทียบแต่ละแนวทางกับเงื่อนไขและข้อจำกัด
5. เลือกแนวทางที่ดีที่สุด (ตอบเงื่อนไขมากที่สุด, ข้อจำกัดน้อยที่สุด)
เครื่องมือออกแบบ: กระดาษ+ดินสอ, โปรแกรม 3D (Tinkercad, SketchUp), แบบจำลอง (Mock-up)
ใช้ตารางตัดสินใจ (Decision Matrix) ช่วยเปรียบเทียบแต่ละแนวทาง
6.4 ขั้นวางแผนและดำเนินการแก้ปัญหา
วางแผนอะไรบ้าง
สร้างต้นแบบคืออะไร
สิ่งที่ต้องวางแผน:
1. ลำดับขั้นตอนการทำงาน (ทำอะไรก่อน-หลัง)
2. วัสดุและเครื่องมือที่ต้องใช้ (รายการ + จำนวน)
3. ระยะเวลาแต่ละขั้น (Gantt Chart หรือตารางเวลา)
4. แบ่งหน้าที่ในทีม (ถ้าทำงานกลุ่ม)
ดำเนินการ: ลงมือสร้างต้นแบบ (Prototype) ตามแผน
ต้นแบบ = ชิ้นงานจริงที่สร้างขึ้นเพื่อทดสอบว่าใช้งานได้จริงหรือไม่
ระหว่างทำบันทึกปัญหาที่เจอ และวิธีแก้ที่ใช้
6.5 ขั้นทดสอบ ประเมินผล และปรับปรุงแก้ไข
ทดสอบอย่างไร
ประเมินผลเทียบกับอะไร
ถ้าไม่ผ่านทำอย่างไร
การทดสอบ:
ทดสอบว่าชิ้นงานทำงานได้ตามเงื่อนไขที่กำหนดไว้ในขั้นที่ 1 หรือไม่
ทดสอบหลายครั้ง, บันทึกผลเป็นข้อมูล (ตัวเลข, ภาพ, วิดีโอ)
การประเมินผล:
เปรียบเทียบผลทดสอบกับเงื่อนไขและข้อจำกัดที่ตั้งไว้
ผ่าน = ตอบเงื่อนไขครบ / ไม่ผ่าน = ยังไม่ตอบเงื่อนไขบางข้อ
การปรับปรุง:
ถ้าไม่ผ่าน → วิเคราะห์ว่าจุดไหนที่ต้องแก้ → วนกลับไปขั้นที่เหมาะสม
อาจแก้ที่วัสดุ, โครงสร้าง, ขนาด, หรือออกแบบใหม่ทั้งหมด
เป็นกระบวนการวนซ้ำ (Iterative): ทดสอบ → ปรับ → ทดสอบ → ปรับ จนผ่าน
6.6 ขั้นนำเสนอวิธีการแก้ปัญหา
นำเสนออะไรบ้าง
นำเสนออย่างไรให้ดี
สิ่งที่ต้องนำเสนอ:
1. ปัญหาที่เลือกแก้ + เงื่อนไข/ข้อจำกัด
2. แนวคิดและแนวทางที่เลือก (ทำไมถึงเลือก)
3. แบบร่าง/แบบจำลอง
4. ขั้นตอนการสร้างชิ้นงาน
5. ผลการทดสอบ + การปรับปรุง
6. ข้อสรุปและข้อเสนอแนะ
รูปแบบนำเสนอ: โปสเตอร์, พรีเซนเทชัน, รายงาน, วิดีโอ
เคล็ดลับ: ใช้ภาพประกอบ, พูดชัด, ตอบคำถามได้, แสดงชิ้นงานจริง
สรุปบทที่ 6
กระบวนการออกแบบเชิงวิศวกรรม 6 ขั้น: ระบุปัญหา (กำหนดเงื่อนไข/ข้อจำกัด) → รวบรวมข้อมูล → ออกแบบ (ร่าง 2-3 แนวทาง เลือกดีสุด) → วางแผนและสร้างต้นแบบ → ทดสอบ/ประเมิน/ปรับปรุง (วนซ้ำได้) → นำเสนอผลงาน เป็นกระบวนการที่ย้อนกลับได้ทุกขั้นจนกว่าจะสำเร็จ
7
กรณีศึกษา: การแก้ปัญหาตามกระบวนการออกแบบเชิงวิศวกรรม
กระบวนการที่ 1: การระบุปัญหา
เลือกปัญหาอย่างไร
เขียนเงื่อนไขอย่างไร
สำรวจปัญหาในชีวิตประจำวัน สิ่งที่ไม่สะดวก ไม่ปลอดภัย ไม่มีประสิทธิภาพ
วิธีหาปัญหา:
สังเกตรอบตัว, สอบถามผู้อื่น, วิเคราะห์กิจกรรมประจำวัน
เขียนข้อความปัญหาให้ชัดเจน:
"นักเรียนต้องวางโทรศัพท์บนโต๊ะเพื่อดูวิดีโอเรียนออนไลน์ แต่โทรศัพท์ล้มบ่อยและปรับมุมไม่ได้"
กำหนดเงื่อนไข: รับน้ำหนักโทรศัพท์ได้ 200g, ปรับมุมได้ 3 ระดับ, ตั้งได้มั่นคง
กำหนดข้อจำกัด: งบ ≤ 80 บาท, เวลา ≤ 3 ชม., วัสดุที่หาได้ในท้องถิ่น
กระบวนการที่ 2: การรวบรวมข้อมูลและแนวคิด
ค้นคว้าอะไรบ้าง
ใช้ข้อมูลอย่างไร
ค้นคว้าข้อมูล:
1. ผลิตภัณฑ์ที่มีอยู่ในท้องตลาดที่คล้ายกัน (ข้อดี/ข้อเสีย)
2. หลักการทางวิทยาศาสตร์ที่เกี่ยว (เช่น จุดศูนย์กล่าว, แรงเสียดทาน)
3. สมบัติวัสดุที่จะใช้ (ไม้อัด, กระดาษแข็ง, พลาสติก)
4. ความต้องการจริงของผู้ใช้ (สัมภาษณ์เพื่อน/ครู)
Brainstorm แนวคิดหลายๆ ทาง ยังไม่ตัดอะไรทิ้ง
จัดกลุ่มแนวคิดที่ใกล้เคียงกัน เพื่อนำไปร่างแบบในขั้นต่อไป
กระบวนการที่ 3: การออกแบบ สร้าง ทดสอบ และนำเสนอ
ร่างแบบอย่างไร
สร้างต้นแบบอย่างไร
ทดสอบและปรับปรุงอย่างไร
นำเสนออย่างไร
ออกแบบ:
ร่างแบบอย่างน้อย 3 แนวทาง พร้อมระบุวัสดุ ขนาด
ใช้ตารางตัดสินใจ (Decision Matrix) เปรียบเทียบกับเงื่อนไข
เลือกแนวทางที่ได้คะแนนสูงสุด
สร้างต้นแบบ:
วางแผนขั้นตอน → จัดเตรียมวัสดุ/เครื่องมือ → ลงมือทำ
คำนึงถึงความปลอดภัย, บันทึกขั้นตอนเป็นภาพ/วิดีโอ
ทดสอบ:
ทดสอบตามเงื่อนไขทีละข้อ เช่น รับน้ำหนักได้ไหม, ปรับมุมได้ไหม, มั่นคงไหม
บันทึกผล: ผ่าน/ไม่ผ่าน แต่ละข้อ
ปรับปรุง:
ไม่ผ่านข้อไหน → วิเคราะห์สาเหตุ → แก้ไข → ทดสอบซ้ำ
นำเสนอ:
สรุปกระบวนการทั้งหมด: ปัญหา → ข้อมูล → แบบ → สร้าง → ทดสอบ → ปรับปรุง → ผลสำเร็จ
แสดงชิ้นงานจริง, เปรียบเทียบก่อน-หลังปรับปรุง
สิ่งสำคัญ: บันทึกทุกขั้นตอน, ทำงานเป็นทีม, รับฟังคำติชม
สรุปบทที่ 7
บทนี้คือ "ภาคปฏิบัติจริง" นำกระบวนการออกแบบเชิงวิศวกรรมทั้ง 6 ขั้นมาใช้แก้ปัญหาจริง ตั้งแต่สำรวจปัญหาในชีวิตประจำวัน กำหนดเงื่อนไข ค้นคว้า ร่างแบบ สร้างต้นแบบ ทดสอบ ปรับปรุง จนนำเสนอ ย้ำว่าต้องบันทึกทุกขั้นตอน ทำซ้ำได้จนสำเร็จ
